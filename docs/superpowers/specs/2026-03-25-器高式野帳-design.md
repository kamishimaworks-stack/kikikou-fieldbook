# 器高式野帳 — 設計書

## 概要

器高式水準測量の野帳をデジタル化するWebアプリ。スマホでの現場入力をメインに、スプレッドシートのようなリアクティブ計算、カスタム数字キーパッド、オフライン対応を備える。

- **技術スタック**: Vanilla HTML/CSS/JS（フレームワークなし）
- **動作環境**: ブラウザ（PWA対応、オフライン可）
- **主な使用端末**: スマホ/タブレット（現場利用）
- **データ保存**: LocalStorage（サーバー不要）

---

## 画面構成

### 1. ファイル一覧画面（ホーム）

- 保存済みファイルの一覧（現場名・作成日・表示モード）
- 「新規作成」ボタン → ダイアログで現場名と表示モード（地盤高m / プラマイmm）を選択
- 各ファイルに対して「開く」「名前変更」「削除」操作
- 一覧は更新日時の降順で表示

### 2. 野帳入力画面（メイン）

**ヘッダー:**
- 現場名表示、「クリア」（全行のデータを消去、確認ダイアログ付き）「保存」「戻る（一覧へ）」ボタン

**テーブル（7列）:**

| 列 | 名称 | 入力 | 背景 |
|----|------|------|------|
| 測点 | 測点名 | テキスト手入力 | 通常 |
| BS | 後視 (m) | 数値手入力 | 通常 |
| IH | 器械高 (m) | 自動計算（入力不可） | ピンク |
| FS | 前視 (m) | 数値手入力 | 通常 |
| GH | 地盤高 (m) | 初回は手入力、以降は自動計算 | 通常 |
| FH | 設計値 (m) | 数値手入力 | 通常 |
| Diff | 差 (m) | 自動計算（入力不可） | ピンク |

**行操作:**
- 下部に「＋行を追加」ボタン
- 各行の間にタップ可能な挿入エリア
- 行の削除（左スワイプまたは長押し）

### 3. カスタム数字キーパッド

```
┌─────────────────────────────┐
│  ＋／－  │  12.345   │ 完了 │
├─────────┼─────────┼───────┤
│    7    │    8    │   9   │
├─────────┼─────────┼───────┤
│    4    │    5    │   6   │
├─────────┼─────────┼───────┤
│    1    │    2    │   3   │
├─────────┼─────────┼───────┤
│    .    │    0    │   ⌫   │
└─────────────────────────────┘
```

- 画面下部からスライドアップで表示
- 「＋／－」ボタンで符号をワンタップ切替
- 「完了」で入力確定 → キーパッド閉じ → 再計算実行
- ネイティブキーボードは抑制（readonly属性 + カスタムキーパッド）
- キーパッド表示中はテーブルがスクロールして対象セルが見える位置に調整

## 計算ロジック

### 基本式

```
IH = GH + BS    （器械高 = 地盤高 + 後視）
GH = IH - FS    （地盤高 = 器械高 - 前視）
Diff = GH - FH  （差 = 地盤高 - 設計値）
```

### リアクティブ計算エンジン

- どのセルが変更されてもテーブル全体を上から順に再計算する
- 行数が少ないため性能問題なし

### 計算ルール

テーブルを上から順に走査し、以下のルールで各行を処理する:

1. **GHの決定**（優先順位順）:
   - manualFieldsに"gh"がある → 手入力値をそのまま使用
   - FSが入力されており、有効なIHが存在する → `GH = IH(現在有効) - FS`
   - 上記のいずれでもない → GHは空白

2. **IHの決定**:
   - BSが入力されており、GHが確定している → `IH = GH + BS`（新しい器械高）
   - BSがない → IHは前の行から引き継がれる（表示しない）

3. **Diffの決定**:
   - GHとFHの両方が値を持つ → `Diff = GH - FH`
   - いずれかが空 → Diffは空白

4. **最初の行のGHは手入力**（既知点の地盤高）。行の挿入・削除で先頭行が変わった場合も、manualFieldsフラグで手入力値は保持される。

### BSとFSが同一行にある場合（移器点）

移器点では1つの行にFSとBSの両方が入力される。計算順序:
1. まず `GH = 旧IH - FS`（旧器械での地盤高を確定）
2. 次に `IH = GH + BS`（新器械の器械高を計算）

この順序により、旧器械の最終測定と新器械の据え付けが同一行で処理される。

### 表示モード

- **地盤高(m)**: BS, IH, FS, GH, FH, Diffすべてをm単位で表示（小数3桁）
- **プラマイ(mm)**: BS, IH, FS, GH, FH, Diffすべてを×1000してmm単位で表示（整数）
- **内部保存は常にm単位**。表示モードは表示のみに影響する
- **入力時**: 地盤高モードではm単位で入力、プラマイモードではmm単位で入力（内部で÷1000してm変換して保存）
- ファイル作成時に選択、画面上でも切替可能

### 手入力セルの視覚的区別

- 各セルに `isManual` フラグを保持
- ユーザーが直接入力 → `isManual = true` → 青いドットまたは青下線を表示
- 計算で自動設定 → `isManual = false` → 通常表示
- IH列・Diff列は常に自動計算（手入力不可）

## データ構造

### 行データ（1行分）

```json
{
  "id": "crypto.randomUUID()で生成",
  "point": "測点名",
  "bs": null,
  "fs": null,
  "gh": null,
  "fh": null,
  "ih": null,
  "diff": null,
  "manualFields": ["gh", "bs"]
}
```

### ファイルデータ

```json
{
  "files": {
    "file-id-1": {
      "id": "file-id-1",
      "name": "現場名A",
      "displayMode": "m",
      "createdAt": "2026-03-25T...",
      "updatedAt": "2026-03-25T...",
      "rows": []
    }
  }
}
```

## 行操作

- **末尾に追加**: 下部の「＋行を追加」ボタン
- **行間挿入**: 各行の下辺にタップ可能な挿入エリア（高さ20px以上、タップ時に「＋」アイコン表示、タップで間に空行を挿入）
- **行削除**: 左スワイプまたは長押しで削除ボタン表示、削除時は確認トースト「行を削除しました [元に戻す]」を表示
- 行の追加・削除後は即座に全体再計算
- 各行はユニークIDを持ち、挿入・削除でもインデックス追跡可能

## UI/UX詳細

### スマホ最適化

- テーブルは横スクロール可能（7列はスマホ幅を超える可能性）
- 測点列は左に固定（sticky）して常に見える
- セルは十分なタップ領域を確保（最低44px高さ）
- 現在編集中のセルはハイライト表示

### 色設計

| 要素 | 色 |
|------|-----|
| ヘッダー | 紺色（#3b5998系） |
| 自動計算列（IH, Diff） | ピンク背景 |
| 手入力済みセル | 左端に青いドットまたは青下線 |
| 行間挿入エリア | タップ時に薄い青線表示 |
| 「＋行を追加」ボタン | 紺色 |
| Diff がマイナス | 赤文字 |
| Diff がプラス | 青文字 |

### エラー・空値の扱い

- 未入力セルは空白表示（0ではない）
- 計算に必要な値が揃わない場合、結果セルも空白
- 不正な入力はキーパッドが数字のみ受け付けるため発生しない

### 保存タイミング

- 「保存」ボタンで明示的に保存
- セル編集確定ごとに自動保存（データロスを防ぐ）
- `visibilitychange` イベントでも保存（モバイルブラウザで `beforeunload` が不安定なため）
